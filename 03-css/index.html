Cascading Style Sheets (CSS)

Cascading Style Sheets (CSS)

  • Tells the browser how to display a certain element
  • Offers lots of power & versatility: CSS Zen Garden
  • Follows the general ruleset:
    1. Select the HTML element to be styled
    2. Specify the properties of the element to be styled
    3. Give the values we want each property to have
Not a programming language like Javascript, nor a mark-up language like HTML
Nothing quite like it
You can think of it like a Style Configuration File
Easy to understand and use but very hard to do it right

Structure of a CSS rule

selector {
    property1: value;
    property2: value;
    property3: value;
}
div {
    color: #fff;
    font-size: 24px;
}
  • The selector identifies which HTML element(s) the style rule will be applied to
  • The curly braces contain the property-value pairs, each separated by a semi-colon
  • The property indicates the style to be changed
  • The value is what the properties are set to for that element

Basic Types of CSS selectors

  • Element: matches all the elements of that name on the page 
    p {}
  • Class: matches all the elements with the specified class, e.g. <div class="example"> 
    .example {}
  • ID: matches the element with the specified id, e.g. <div id="example"> 
    #example {}

More Types of CSS selectors

  • Attribute: matches all the elements with provided attribute, e.g. <video controls> or <a href="https://google.com"> 
    [controls] {}
    [href="https://google.com"] {}
  • Combination: matches all the elements with the specified combination of selectors, e.g. <div class="wrapper"> 
    div.wrapper {}
    notice there is no space between this combination selector

Even More Types of CSS selectors

  • Nested: matches the nested descendant element with the specified ancestor 
    <ul>
  <h1>Title</h1>
  <li>
    <a href="/link">Link</a>
  </li>
  <li>
    <p>
      <a href="/hello">Hello</a>
    </p>
  </li>
  <li>item</li>
</ul>
    ul li {}
    ul a {}
    ul > li {}
    ul > li > a {} // this only matches the first link
    ul > a {} // this selector won't match anything above ðŸš«
  • Sibling Selectors: matches subsequent sibling or siblings (no back-traversal) 
    h1 + li {} // selects first suubsequent li element after h1
    h1 ~ li {} // selects all subsequent li elements after h1

CSS Triggers and the CSSOM

                

                
                

                    
How browsers render stuff 

                    visualized
                    
    
                        
  • Different browsers use different rendering engines but follow similar flow
    • 
                        
  • Browser will parse the markup, then "Recalculate Style" to figure out what styles apply to which elements
                        
    • 
                        
  • Next, it will "Layout" all the DOM elements
    • 
                        
  • Then, "Paint" the pixel data for the elements
    • 
                        
  • And, "Composite" will combine and draw the painted layers out onto the screen
    • 
                    

                

                

                    
Rendering: HTML and CSS Parsers

                    
                    
Browser creates the DOM and the CSSOM

                

                

                    
The DOM

                    
                

                

                    
The CSSOM

                    
                

                

                    
The Render Tree

                    
                    
But if you change anything in the DOM or CSSOM, the Render Tree must be recalculated

                

                

                    
Think of it as a pyramid

                    
                    
If you change something at the bottom, the whole pyramid has to be rebuilt.

                    
If you change just the top layer, the layers beneath remain intact.

                

                
                    

                        
Layout

                        
    
                            
  • Generate geometry and position of each element
    • 
                            
  • Very expensive because engine must reflow the relevant element and all its children to take into account
                                the new layout
    • 
                            
  • Occurs when the DOM tree is manipulated, or a style which affects layout is changed, when browser window
                                size is changed, when className property is changed
    •

Paint

                        
    
                            
  • Fill out pixels for each element into layers
    • 
                            
  • Make something visible when it previously was not, like adding outline or changing background colour
                            
    • 
                            
  • Expensive because the engine searches through all elements to determine what is visible and what should
                                be displayed
    • 
                            
  • Browser vendors are scrambling to improve the renderer, but until recently browsers weren't built to
                                handle animations efficiently
    •

Composite

                        
These are safe to dynamically change and animate.

                        
    
                            
  • Only Opacity and Transforms trigger composite
    • 
                            
  • Blink and Webkit creates new layer for elements with CSS transforms or opacity
    • 
                            
  • translateZ(0) is used to force layer creation which ensures both the layer is painted as soon as the
                                animation starts
    • 
                            
  • Watch The Web's Black Magic for best explanation
                                on rendering and compositing
